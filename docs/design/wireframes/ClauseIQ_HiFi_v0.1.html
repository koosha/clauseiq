What's next

Interactive prototype

Next deliverable: a working click-through built on these mockups. Real keyboard shortcuts, real pinning, real drill-in, palette fully functional. Fake but realistic clause data. Ready to demo and to user-test with 2–3 in-house lawyers.

Success criteria: median time-to-first-pin under 60 seconds; queries per session ≥ 6 in a 15-minute task; zero users asking "where did my pinned clauses go?"

Still open

  • Scratchpad scope. Global vs per-matter. I lean per-matter; the dock would then show the current matter name.
  • Citation style in DOCX. Word comments vs footnotes vs stripped. I lean comments — they delete cleanly.
  • Family labels. Pretty ("Limitation of liability") or token ("limitation_of_liability"). I lean pretty for the main UI, token in the debug block.
  • Result count. 20 is the API cap; I show 14 here after filters. Should we paginate if fewer filters return 20? I lean no; the reranker cuts it down.
